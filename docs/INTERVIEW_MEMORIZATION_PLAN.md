# 3天背诵计划（控场版）

> 目标：尽量把面试问题“引导”到你已准备的内容里，降低被问到文档外细节的概率。  
> 注意：没人能 100% 控制面试官，但你可以通过回答结构和追问引导，大幅提高命中率。

---

## 使用规则（先背这个）

1. **所有回答都走固定模板**：  
   `一句结论 -> 两个关键点 -> 项目落地例子 -> 可扩展一句`

2. **先讲你熟的，再补你不熟的**：  
   比如被问“分布式锁”，先说你项目实际用的是 `Lua + DB 条件更新 + 补偿`，再说“如果上量可以补 Redisson 方案”。

3. **遇到超纲问题，永远往项目拉**：  
   “这个我在当前项目没有上线，但我可以结合我们秒杀链路说明我会怎么做……”

4. **不要硬说不会**，改说：  
   “我当前实践重点在 A/B/C，这块我有方案思路，落地优先级会放在……”

---

## Day 1：必会基础（高频 + 稳分）

### 上午（2小时）
- 项目全景：架构、模块、调用链（前端页面 -> api -> controller -> service）
- Java 基础：`synchronized`、`Atomic`、异常分层
- Spring 基础：IOC、AOP、拦截器、全局异常处理

### 下午（2小时）
- MySQL：索引、事务、主子表设计（order/order_item）
- Redis：缓存、Session、限流
- 前端：Vue3、Pinia、Router 守卫、Axios 封装

### 晚上（1.5小时）
- 口头默写 20 题（从 `INTERVIEW_QA.md` 前 40 题挑）
- 每题严格 60~90 秒

---

## Day 2：项目核心（秒杀 + 一致性 + 可观测）

### 上午（2小时）
- 秒杀链路完整时序（path -> 限流 -> Lua -> DB -> 补偿）
- 为什么 Lua、为什么 DB 兜底、为什么还要监控

### 下午（2小时）
- 监控与定时任务：QPS、库存偏差、超时取消
- AI 模块：实时快照注入、角色视图、fallback

### 晚上（1.5小时）
- “追问反杀”专项（`INTERVIEW_QA.md` 第18章）
- 重点背 10 个：Redis 挂了、慢查询、超卖证明、MQ 取舍、幂等

---

## Day 3：控场演练（模拟面试）

### 上午（1.5小时）
- 3分钟自我介绍（背熟）
- 8分钟项目讲解（背熟）
- 30秒收尾话术（背熟）

### 下午（2小时）
- 模拟面试 2 轮（每轮 30 分钟）
- 复盘：每个问题写“我如何把它拉回项目”

### 晚上（1小时）
- 最后记忆压缩：每章保留 3 句关键词

## 高频问题“控场桥接句”（直接背）

### 1) 被问到陌生技术细节
“这个点我项目里不是主路径，我主路径是 **Lua 原子扣减 + DB 条件更新 + 补偿**。如果要扩展到你说的方案，我会这样做……”

### 2) 被问到太底层原理
“我先讲业务上为什么要这么设计，再补底层原理：我们当时核心目标是防超卖和可观测……”

### 3) 被问到你没上线过的内容
“线上我还没全量落地，但我已经有分阶段方案：先做 A，再做 B，最后做 C。”

### 4) 面试官打断你
“明白，我用一句话总结：**当前方案解决了 X，代价是 Y，下一步是 Z**。”

---

## 你的“安全回答边界”

优先回答以下你已准备区：
- 秒杀一致性（Lua、DB、补偿）
- 登录鉴权（Session + 拦截器 + 管理员路由）
- 前后端调用链（views/api/controller/service）
- 监控与定时任务
- AI 实时快照注入

遇到文档外问题，统一策略：
- 先承认“当前项目没上线该特性”
- 立即给“可落地方案 + 优先级”
- 回到你的项目事实
